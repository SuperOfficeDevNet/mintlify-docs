## Implementation Strategy Recommendations

### Option A: Full Migration (Comprehensive)
- Add all 60+ nested groups systematically
- Maintains complete documentation structure
- **Estimated effort**: 50-70 file replacements in docs.json
- **Time**: 30-45 minutes
- **Best for**: Complete documentation portal

### Option B: Priority-Based Migration (Incremental)
- Phase 1: High-priority sections (Getting Started, Document, Email, Marketing)
- Phase 2: Medium-priority sections (Sale, Project, Request, Customization)
- Phase 3: Lower-priority sections (Dashboard, Chat, etc.)
- **Estimated effort**: 15-20 replacements per phase
- **Best for**: Iterative deployment

### Option C: Essential Only (Minimal)
- Add only the most critical user-facing features
- Getting Started (2 groups), Document Templates (2 groups), Marketing (5 groups), Admin essentials (3 groups)
- **Estimated effort**: ~15-20 replacements
- **Time**: 15-20 minutes
- **Best for**: Quick launch with core features

## Technical Notes
1. **Inline vs Separate toc.yml**: Some groups (like Sale > Stakeholders) use inline items arrays rather than separate toc.yml files
2. **Cross-References**: Customization section references automation/crmscript, email/service, customer-center, ui/blogic
3. **Admin Sections**: Often deeply nested (3-4 levels) with multiple inline groups
4. **Consistent Patterns**: Most sections follow: Learn section → nested features → admin section
5. **Language Parity**: Same structure should exist for sv, nl, da languages
